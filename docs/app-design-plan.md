# 桌面布局工坊 App 端设计规划

## 目标

把原本依赖电脑脚本完成的事情迁移到手机 App 内：

- 分组规则：分类名、名称关键词、包名关键词、规则顺序。
- 排版策略：分类优先级、偏好页面、文件夹占格大小。
- 布局生成：读取已保存布局，保留组件占位，重建文件夹，生成新的可恢复 JSON。
- 日常管理：导入、导出、改名、删除、多选、查看结构、复制报告。
- 正常 App 能力：多页面导航、弹窗/底部菜单、暗色/亮色/跟随系统、中文/英文界面、诊断信息。

## 页面结构

### 布局库

负责布局文件生命周期：

- 保存当前桌面。
- 导入 JSON 布局。
- 恢复布局到系统桌面。
- 改名、删除、导出副本、查看结构。
- 多选、全选、批量删除。
- 左右滑动打开操作菜单。

正常情况下不显示“桌面是否连接”等状态；只有失败时显示可复制的错误详情。

### 设计

负责把一个已保存布局重新整理为新布局：

- 选择工作布局。
- 查看布局结构概览。
- 在桌面网格上查看组件占位和规则分组占位。
- 左右滑动切换桌面页面，画布只显示当前页，避免把多个桌面页纵向堆在一起。
- 长按分组或组件后，如果直接松手，就打开编辑面板。
- 长按分组或组件后，如果继续拖动，就把它放到指定页面和格子；只有拖动可以挤开其他分组或组件。
- 分组编辑面板可以调整文件夹大小、编辑规则或进入组内应用顺序管理。
- 改尺寸不会挤开其他项目；目标尺寸如果和已有分组/组件重叠，就要求先拖动重新排布。
- 组件拖动到指定页面和格子后，位置会用于下一次整理生成。
- 拖动时，如果目标位置和其他分组/组件重叠，当前拖动项保留目标位置；它原来的位置会先释放，被挤开的项目优先尝试进入这个空位，再进入后续空位或下一页。
- 使用当前规则整理并保存为新 JSON。
- 显示整理报告，可复制。
- 整理后可以直接恢复，也可以回到布局库继续管理。

整理器会保留桌面 `WIDGETS` 和 `CARD` 占用的格子，只重排应用和快捷方式。

### 规则

负责维护可复用规则：

- 添加、编辑、删除分类规则。
- 上移/下移规则，控制优先匹配顺序。
- 编辑分类名称。
- 编辑名称关键词和包名关键词。
- 编辑文件夹宽度、高度、优先级、偏好页面。
- 扫描手机可启动应用列表。
- 查看还没有命中任何规则的应用，并把它作为精确包名规则加入指定分类。
- 查看某个分类命中的应用。
- 在某个分类内添加新的已安装应用。
- 手动上移/下移分类内应用顺序。
- 按应用名称自动排序分类内应用。
- 重置为 APK 内置默认规则。

规则保存到 App 私有目录，默认规则打包在 APK 的 `assets/rules/launcher_app_categories.json`。

### 设置

负责全局体验和维护：

- 主题：跟随系统、亮色、暗色。
- 语言：跟随系统、简体中文、English。
- 打开规则页或重置规则。
- 复制诊断信息。
- 关于应用。

## 数据流

1. LSPosed 模块仍然只负责和系统桌面进程通信：导出、检查、恢复。
2. App 的 `LayoutStoreProvider` 继续保存布局 JSON 和用户可读名称。
3. `RuleStore` 从 APK 资产加载默认规则，并把用户修改后的规则写入 App 私有目录。
4. `RuleSet` 在 App 内完成分类匹配。
5. `DeviceAppScanner` 通过系统包管理器读取可启动应用列表，用于发现未分组应用和维护组内顺序。
6. 用户手动加入分类的应用会写入 `equals.package`，分类匹配会先处理精确包名，再处理普通规则，避免宽泛关键词抢走手动归属。
7. `LauncherLayoutOrganizer` 在 App 内完成布局解析、分组、文件夹创建、页面占位、组件位置覆盖、组内排序和报告生成。
8. 生成的新布局重新写回 `LayoutStoreProvider`，之后走原来的恢复流程。

## 与电脑脚本的关系

`tools/launcher_layout_editor.py` 和 `tools/rule_engine.py` 仍然保留，作为批处理和调试工具；但日常分组、排版、文件夹大小和规则调整已经不需要电脑脚本。App 内的 Java 版规则引擎和整理器是主流程。
